# Отчёт о доработке проекта «Расписание мероприятий в Крыму»

## Введение

В рамках доработки веб-приложения были решены следующие задачи:

- Исправлен вывод результатов поиска – вместо сырого JSON пользователь видит структурированную HTML-страницу.
- Реализован механизм фильтрации с отображением в модальном окне без перезагрузки страницы.
- Настроен импорт событий через API с проверкой аутентификации.
- Устранены ошибки, связанные с отсутствием классов и несоответствием типов данных.

---

## 1. Изменение логики отправки формы

**Проблема:**
Форма в `index.php` отправляла запрос на `events.php`, который возвращает JSON. Браузер показывал сырые данные вместо читаемой страницы.

**Решение:**
В `index.php` изменён атрибут `action` формы с `events.php` на `eventsget.php`, который формирует HTML-список событий.

**Изменённый фрагмент:**

```html
<form action="eventsget.php" method="get">
```

---

## 2. Устранение ошибки «Class FilterBuilder not found»

**Проблема:**
`eventsget.php` использовал ручные `require_once` для `EventRepository` и `db.php`, но класс `FilterBuilder`, который используется внутри репозитория, не подключался.

**Решение:**
Переход на автозагрузку Composer. В `eventsget.php` заменены ручные подключения на `require_once __DIR__ . '/../vendor/autoload.php';`. Это обеспечивает автоматическую загрузку всех классов пространства имён `App`.

**Итоговый код начала файла:**

```php
<?php
declare(strict_types=1);
require_once __DIR__ . '/../vendor/autoload.php';
use App\EventRepository;
```

---

## 3. Отображение результатов в модальном окне (без открытия новой вкладки)

**Проблема:**
По требованию пользователя результаты должны появляться в модальном окне, а не в новой вкладке.

**Решение:**
В `index.php` убран `target="_blank"` и перехват отправки формы через AJAX. Данные собираются, отправляются на `eventsget.php`, и полученный HTML вставляется в диалоговое окно jQuery UI.

**Ключевой фрагмент скрипта:**

```javascript
$('form').on('submit', function(e) {
    e.preventDefault();
    var formData = $(this).serialize();
    $.ajax({
        url: 'eventsget.php',
        data: formData,
        success: function(response) {
            $('<div>').html(response).dialog({
                modal: true,
                width: '80%',
                title: 'Результаты поиска'
            });
        }
    });
});
```

## 4. Исправление ошибки TypeError при выводе результатов

**Проблема:**
В `eventsget.php` использовался `htmlspecialchars($value ?? '')`, но поле `id` является целым числом, что вызывало `TypeError` в PHP 8+.

**Решение:**
При выводе значения явно приводим его к строке: `htmlspecialchars((string)($value ?? ''))`. Булевы поля обрабатываются отдельно через тернарный оператор.

**Исправленный фрагмент:**

```php
if (is_bool($value)) {
    echo $value ? 'Да' : 'Нет';
} else {
    echo htmlspecialchars((string)($value ?? ''));
}
```

## 5. Настройка импорта данных через `import.php`

**Проблема:**
JSON, переданный в `import.php`, содержал поля `description` и `time`, которых нет в таблице `events`. Это приводило к ошибке выполнения хранимой процедуры.

**Решение:**
Удалены лишние поля из JSON с помощью `jq` перед отправкой:

```bash
jq 'map(del(.description, .time))' events.json > events_clean.json
```

**Проверка синтаксиса JSON:**

```bash
jq . events_clean.json > /dev/null && echo "OK" || echo "Invalid JSON"
```

## 6. Успешный импорт событий

**Команда для отправки:**

```bash
curl -X POST http://192.168.1.36:8099/import.php \
  -H "Content-Type: application/json" \
  -H "X-API-Key: your_secret_key_here" \
  --data-binary @events_clean.json
```

**Ответ сервера:**

```json
{
    "status": "ok",
    "imported": 100,
    "errors": []
}
```

Все 100 событий были успешно добавлены в базу данных.

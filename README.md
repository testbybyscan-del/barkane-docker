# Расписание мероприятий в Крыму

Веб-приложение для поиска и фильтрации культурных, спортивных и развлекательных событий на полуострове Крым. Позволяет пользователям выбирать даты, место, тематику, тип мероприятия и дополнительные опции (для детей, бесплатно, с фуршетом и т.д.). Результаты отображаются в модальном окне без перезагрузки страницы. Также предусмотрен API для массового импорта событий через JSON.

---

## Технологический стек

- **Backend**: PHP 8+ (строгая типизация, PSR-4 автозагрузка)
- **База данных**: PostgreSQL 15+
- **Веб-сервер**: Nginx (в составе Docker-контейнера)
- **Фронтенд**: HTML5, CSS3, JavaScript (jQuery, jQuery UI)
- **Контейнеризация**: Docker + Docker Compose
- **Управление зависимостями**: Composer

## Установка и запуск

### Требования

- Docker Engine 20.10+
- Docker Compose 2.0+
- Git

### Шаги

1. **Клонировать репозиторий**

   ```bash
   git clone https://github.com/your-username/barkane-docker.git
   cd barkane-docker
   ```
2. **Создать файл `.env`** (на основе шаблона)

   ```env
   POSTGRES_DB=barkane
   POSTGRES_USER=barkane
   POSTGRES_PASSWORD=StrongPassword123!
   IMPORT_API_KEY=your_secret_key_here
   DB_HOST=db
   DB_PORT=5432
   ```
3. **Запустить контейнеры**

   ```bash
   docker-compose up -d --build
   ```
4. **Установить зависимости Composer внутри контейнера**

   ```bash
   docker exec -it barkane-php composer install
   ```
5. **Выполнить миграции БД** (если не выполняются автоматически)

   - Скрипты инициализации лежат в `init-db/` и применяются при первом запуске PostgreSQL.
6. **Открыть приложение в браузере**

   ```
   http://localhost:8099
   ```

   (порт может быть изменён в `docker-compose.yml`)

## Использование

### Поиск событий

- На главной странице (`index.php`) заполните поля фильтров.
- **Диапазон дат** обязателен для заполнения (выбирается через календарь).
- Нажмите «Применить фильтры».
- Результаты появятся в модальном окне jQuery UI без перезагрузки страницы.

### Доступные фильтры


| Поле       | Тип               | Описание                                                        |
| -------------- | -------------------- | ----------------------------------------------------------------------- |
| `start_date`   | дата             | Начало диапазона (формат`dd-mm-yyyy`)              |
| `end_date`     | дата             | Конец диапазона                                           |
| `city`         | выпадающий | Топоним Крыма (или «повсюду»)                   |
| `venue`        | выпадающий | Место проведения (или «любое»)                 |
| `theme`        | выпадающий | Тематика (или «любая»)                                |
| `event_type`   | выпадающий | Тип мероприятия (или «любые»)                   |
| `for_kids`     | чекбокс       | Только для детей                                          |
| `is_free`      | чекбокс       | Бесплатные                                                    |
| `has_food`     | чекбокс       | С фуршетом                                                     |
| `has_alcohol`  | чекбокс       | С алкоголем (взаимоисключается с`for_kids`) |
| `has_transfer` | чекбокс       | С трансфером                                                 |

### Пример GET-запроса (вручную)

```
http://localhost:8099/eventsget.php?start_date=01-07-2026&end_date=10-07-2026&city=Ялта&venue=на+берегу+моря&theme=музыка&event_type=концерт&for_kids=on
```

## API для импорта событий

### Эндпоинт: `POST /import.php`

**Заголовки:**

- `Content-Type: application/json`
- `X-API-Key: <ваш_секретный_ключ>` (должен совпадать с `IMPORT_API_KEY`)

**Тело:** массив объектов событий. Каждый объект должен содержать поля:


| Поле       | Тип  | Обязательное | Значение по умолчанию |
| -------------- | ------- | ------------------------ | ---------------------------------------- |
| `name`         | string  | Да                     | –                                       |
| `venue`        | string  | Да                     | –                                       |
| `theme`        | string  | Да                     | –                                       |
| `event_type`   | string  | Да                     | –                                       |
| `city`         | string  | Да                     | –                                       |
| `date`         | string  | Да                     | – (формат`YYYY-MM-DD`)            |
| `for_kids`     | boolean | Нет                   | `false`                                  |
| `is_free`      | boolean | Нет                   | `false`                                  |
| `has_food`     | boolean | Нет                   | `false`                                  |
| `has_alcohol`  | boolean | Нет                   | `false`                                  |
| `has_transfer` | boolean | Нет                   | `false`                                  |

Поля `id`, `description`, `time` **не используются** и будут проигнорированы (если переданы – вызовут ошибку).

**Пример запроса через cURL:**

```bash
curl -X POST http://localhost:8099/import.php \
  -H "Content-Type: application/json" \
  -H "X-API-Key: your_secret_key_here" \
  --data-binary @events_clean.json
```

**Пример успешного ответа:**

```json
{
  "status": "ok",
  "imported": 100,
  "errors": []
}
```

**При неудаче:** `401 Unauthorized` (неверный ключ) или `400 Bad Request` (невалидный JSON), `500` (внутренняя ошибка БД).

## База данных

### Таблица `events`


| Колонка | Тип       | Описание              |
| -------------- | ------------ | ----------------------------- |
| `id`           | SERIAL       | Первичный ключ   |
| `name`         | VARCHAR(255) | Название              |
| `venue`        | VARCHAR(100) | Место                    |
| `theme`        | VARCHAR(100) | Тематика              |
| `event_type`   | VARCHAR(100) | Тип                        |
| `city`         | VARCHAR(100) | Город/регион       |
| `date`         | DATE         | Дата проведения |
| `for_kids`     | BOOLEAN      | Детское                |
| `is_free`      | BOOLEAN      | Бесплатно            |
| `has_food`     | BOOLEAN      | Фуршет                  |
| `has_alcohol`  | BOOLEAN      | Алкоголь              |
| `has_transfer` | BOOLEAN      | Трансфер              |

### Хранимая функция `import_events`

Выполняет «upsert» (вставку или обновление по `id`, если передан). Принимает JSONB и возвращает статус, количество импортированных записей и список ошибок.

## Разработка и тестирование

### Логирование

- Ошибки PHP записываются в `php-fpm` логи (можно посмотреть через `docker logs barkane-php`).
- Ошибки БД логируются в PostgreSQL (контейнер `db`).

### Добавление новых фильтров

1. Добавить поле в форму `index.php`.
2. Расширить массив `$filters` в `eventsget.php` и `events.php`.
3. В `FilterBuilder` добавить метод `applyNewField()` и вызвать его в `build()`.

### Стилизация

- Основные стили – `css/styles.css`.
- Адаптивная вёрстка (медиа-запрос для экранов шире 600px).

## Примечания

- Проект использует **jQuery UI** для календарей и модальных окон; убедитесь, что CDN-ссылки доступны.
- Анимация логотипа (`rotation.js`) перекрашивает и изменяет размер букв каждые 800 мс.
- Чекбоксы «Для детей» и «С алкоголем» взаимно исключают друг друга (логика в `check.js`).
